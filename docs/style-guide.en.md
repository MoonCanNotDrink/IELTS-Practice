# Documentation Style Guide

## Goal

This guide defines the shared rules for navigation style, heading hierarchy, and terminology across the project documentation.

## Naming Rule

- Chinese documents use unsuffixed filenames, such as `README.md`
- English documents use the `.en.md` suffix, such as `README.en.md`

## Navigation Rule

- entry documents should expose a `Navigation` section near the top
- links should default to same-language targets
- if a counterpart exists, provide an explicit cross-language link
- user-guide documents should use `Continue Reading` at the bottom

## Heading Hierarchy

- each document should have exactly one `#` title
- primary sections should use `##`
- secondary sections should use `###`
- avoid skipping heading levels

## Preferred English Terms

Use the following terms consistently:

- `Home`
- `Documentation Index`
- `Project Overview`
- `External Documentation`
- `User Guide`
- `Getting Started`
- `Mock Exam Guide`
- `Free Practice Guide`
- `FAQ`
- `Continue Reading`

## Preferred Chinese Terms

When referring to Chinese pages, prefer these labels:

- `完整模考`
- `自由练习`
- `题库`
- `文档目录`
- `项目简介`
- `对外文档`
- `用户指南`
- `快速开始`
- `常见问题`
- `继续阅读`
