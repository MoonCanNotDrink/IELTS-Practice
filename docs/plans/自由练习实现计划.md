# 自由练习实现计划

## 目标

为项目增加一个自由练习模式，使用户可以：

1. 输入自定义口语题目；
2. 从预设时长中选择作答倒计时，或输入自定义秒数；
3. 直接开始作答；
4. 复用现有的转写、上传与评分链路完成整套流程。

目标是在尽量少改动现有系统结构的前提下，扩展出一个低门槛、可重复使用的练习入口。

## 约束

- 复用现有 Part 2 上传与评分链路。
- 保持改动最小，避免无关重构。
- 除非绝对必要，不进行数据库结构变更。
- 不影响现有 Full Exam 与 Part 2 Only 的行为。
- 前端风格应与当前应用保持一致。

## 当前可复用能力

- 前端状态机与 Part 2 流程位于 `frontend/app.js`。
- 主界面结构位于 `frontend/index.html`。
- 现有样式位于 `frontend/index.css`。
- Part 2 会话创建、上传与评分逻辑位于 `backend/app/routes/part2.py`。
- 历史记录与详情回退逻辑位于 `backend/app/routes/scoring.py`。
- 现有 Part 2 评分已经使用 `Recording.question_text`，可承载自由练习题目。

## 实施方案

### 1. 后端支持自定义题目会话

- 扩展 Part 2 会话创建逻辑，使其既支持 `topic_id`，也支持用户传入 `custom_topic`。
- 保持 `PracticeSession.topic_id` 可为空，录音类型仍按 `part2` 存储。
- 在上传阶段，当会话没有预置题目时，将用户自定义题目写入 Part 2 的 `question_text`。

### 2. 历史记录与详情支持自由练习题目回退

- 更新 history/detail 返回逻辑，使已完成的自由练习会话优先展示已保存的 Part 2 `question_text`，而不是 `Unknown`。
- 保持评分范围仍为 `part2_only`，以便现有结果页继续复用。
- 在评分阶段，当会话没有 Topic 实体时，从已保存的 Part 2 录音中回填 `question_text`，保证提示词上下文正确。

### 3. 前端增加自由练习入口与配置表单

- 在首页增加自由练习模式入口。
- 增加一个轻量配置表单，包含：
  - 自定义题目输入框；
  - 预设时长按钮；
  - 自定义秒数输入框。
- 校验题目是否为空，以及自定义时长是否为正整数。

### 4. 前端复用 Part 2 作答与评分流程

- 复用现有的题卡展示、录音、上传、转写与评分页面。
- 尽可能把自由练习题目映射为现有题卡结构，避免分叉 UI。
- 将作答计时器改为可配置，而不是固定 120 秒。

### 5. 测试与验证

- 增加后端测试，验证自定义会话创建与 history/detail 回退逻辑。
- 增加 Playwright 测试，覆盖自由练习前端路径。
- 扩展前端静态测试服务器，使其可以 mock 新的自由练习 API 请求。
- 运行完整验证命令，确保新老流程都稳定。
- 手动在本地浏览器跑通一遍完整自由练习链路。

## 验收标准

- 用户无需抽题即可开始一轮自由练习。
- 用户可以选择预设时长，也可以输入自定义秒数。
- 作答倒计时会正确使用用户选择的时长。
- 作答完成后，系统继续复用现有转写和评分流程。
- 已完成的自由练习会话可在历史记录与详情中展示真实题目。
- 现有 Full Exam 与 Part 2 Only 流程继续通过测试。

## 验证命令

- `cd backend && (python -m unittest test_tts_route test_free_practice_routes -v || python3 -m unittest test_tts_route test_free_practice_routes -v)`
- `npx playwright test`
- 本地浏览器手动验证自由练习链路，重点确认：
  - 输入题目与时长后能进入自由练习；
  - 倒计时使用正确时长；
  - 录音完成后进入现有评分页；
  - 历史记录与详情页正确展示自由练习题目。
