# Free Practice Implementation Plan

> Written for project version: v0.2.x (pre-refactor). Last recorded in git: 2026-03-20.
>
> Note: The frontend has since been restructured into multiple modules and separate page scripts. See docs/superpowers/specs/2026-03-31-frontend-modularization-and-pattern-standardization-design.md for the current frontend architecture. This plan remains as a historical implementation plan and may not match the current file layout.

## Goal

Add a Free Practice mode that lets users:

1. enter a custom speaking prompt;
2. choose a preset speaking duration or enter a custom number of seconds;
3. start answering directly;
4. reuse the existing upload, transcription, and scoring flow end to end.

The goal is to extend the current system with a low-friction practice entry while keeping structural changes as small as possible.

## Constraints

- Reuse the existing Part 2 upload and scoring pipeline.
- Keep the diff small and avoid unrelated refactoring.
- Avoid database schema changes unless they are strictly necessary.
- Do not break the existing Full Exam or Part 2 Only flows.
- Keep the frontend style aligned with the current product.

## Reusable Capabilities Already Present

- The frontend state machine and Part 2 flow live in `frontend/app.js`.
- The main UI structure lives in `frontend/index.html`.
- Existing styles live in `frontend/index.css`.
- Part 2 session creation, upload, and scoring live in `backend/app/routes/part2.py`.
- History/detail fallback logic lives in `backend/app/routes/scoring.py`.
- Existing Part 2 scoring already uses `Recording.question_text`, which can store a Free Practice prompt.

## Implementation Plan

### 1. Support custom-topic sessions in the backend

- Extend Part 2 session creation so it accepts either `topic_id` or `custom_topic`.
- Keep `PracticeSession.topic_id` nullable while still storing the recording as `part2`.
- During upload, if a session has no predefined topic, persist the custom prompt into Part 2 `question_text`.

### 2. Support Free Practice topic fallback in history and detail views

- Update history/detail responses so completed Free Practice sessions prefer the saved Part 2 `question_text` instead of `Unknown`.
- Keep the scoring scope as `part2_only` so the existing result page can be reused.
- During scoring, if a session has no Topic entity, fill prompt context from the saved Part 2 recording.

### 3. Add a Free Practice entry and setup form in the frontend

- Add a dedicated Free Practice entry on the homepage.
- Add a lightweight setup form with:
  - a custom prompt field;
  - preset duration buttons;
  - a custom seconds input.
- Validate that the prompt is not empty and that custom duration is a positive integer.

### 4. Reuse the existing Part 2 answering and scoring flow

- Reuse the current card display, recording, upload, transcription, and scoring screens.
- Map the Free Practice prompt into the existing topic-card structure where possible.
- Make the speaking timer configurable instead of fixed at 120 seconds.

### 5. Test and validate

- Add backend tests for custom session creation and history/detail fallback logic.
- Add Playwright coverage for the Free Practice frontend path.
- Extend the frontend test server so it can mock the new Free Practice API.
- Run the full verification commands to ensure both old and new flows stay stable.
- Manually test the full Free Practice flow in a browser.

## Acceptance Criteria

- Users can start a Free Practice session without drawing a topic.
- Users can choose either a preset duration or a custom duration.
- The speaking timer uses the user-selected duration.
- After answering, the system reuses the existing transcription and scoring flow.
- Completed Free Practice sessions show the real prompt in history/detail views.
- Existing Full Exam and Part 2 Only flows continue to pass tests.

## Verification Commands

- `cd backend && (python -m unittest test_tts_route test_free_practice_routes -v || python3 -m unittest test_tts_route test_free_practice_routes -v)`
- `npx playwright test`
- Manual browser verification of the Free Practice flow, especially:
  - entering a prompt and duration reaches Free Practice;
  - the timer uses the correct duration;
  - recording completion reaches the existing scoring page;
  - history and detail pages show the real Free Practice prompt.
