# 自由练习界面优化报告

返回目录：[文档目录](../README.md) | English version: [Free Practice UI Optimization Report](自由练习界面优化报告.en.md) | 上一篇：无 | 下一篇：[自由练习-产品设计摘要](自由练习-产品设计摘要.md)

## 背景

本报告总结了 Free Practice 页面这一轮界面审查与优化工作的完整结果，重点覆盖以下区域：

- `Topic Library` 与 `Write My Own` 的模式切换
- Topic Library 下拉选择器
- 题目搜索、空状态与加载状态
- 可读性与状态辨识度
- 键盘可访问性与 ARIA 语义增强

本轮工作的起点是一个明确的用户体验问题：原有 Topic Library 选择方式不够易用，尤其在话题较多、文本较长时，选择过程效率低，界面反馈也不够清晰。随着实现推进，这次优化进一步扩展为对整个 Free Practice 输入区的可读性、状态表达和可访问性的一次系统修复。

## 发现的问题

### 1. 题目选择效率低

- 原生选择体验不适合较长的话题列表。
- 用户无法快速搜索或过滤题目。
- 长文本题目在选择前不容易完整阅读。

### 2. 用户意图区分不清晰

- `Topic Library` 与 `Write My Own` 代表的是两种不同的输入路径。
- 旧界面没有把这两种意图区分得足够明确。

### 3. 视觉状态辨识度不足

- 下拉框文字与背景层次过近，阅读压力较大。
- 模式切换按钮的选中态不够明显。
- placeholder、空状态等次级文本在早期实现中可读性不足。

### 4. 可访问性支持不完整

- 自定义下拉框早期主要依赖鼠标操作。
- 键盘导航路径缺失。
- trigger、listbox、search input 的语义信息不足。
- 重新打开下拉框时，选中项的视觉状态与语义状态一度可能不同步。
- 视觉分组标题最初只是视觉提示，并没有真正暴露给辅助技术。

## 设计目标

本轮优化遵循四个原则：

1. 明确区分用户输入意图；
2. 提升题目浏览、搜索与扫描效率；
3. 在不破坏现有视觉体系的前提下增强对比度与状态层次；
4. 让自定义下拉框具备可接受的键盘与辅助技术可访问性基础。

## 已完成的优化

### 1. 引入更清晰的模式切换控件

Free Practice 入口区现在使用独立的分段切换控件来区分：

- `Topic Library`
- `Write My Own`

这样可以让用户在开始前立即理解当前是“从题库选题”还是“自行输入题目”，减少认知负担。

### 2. 用自定义下拉框替代基础选择器

新的 Topic Library 选择器支持：

- 分组展示
- 实时搜索
- 更适合长文本阅读的选项布局
- 选中状态保留
- 加载态与空状态反馈

当前话题被分为两组：

- `Official Topics`
- `Your Saved Topics`

### 3. 强化整组控件的可读性与视觉层次

本轮重新调整了以下区域的对比关系：

- 模式切换容器
- 非激活与激活按钮
- 下拉触发器
- 搜索输入框
- 分组标题
- 普通选项
- hover 选项
- selected 选项
- 加载与空状态

最终选中态采用了更明确的蓝色强调、更稳定的亮色文字和更清晰的深色背景层次，使这组控件与当前应用整体暗色视觉体系保持一致，同时显著提升可辨认性。

### 4. 完成键盘可访问性增强

当前下拉框支持：

- `Enter` 从触发器打开
- `ArrowDown` 从触发器打开
- `ArrowUp` / `ArrowDown` 在选项间移动
- `Enter` 或 `Space` 选择当前焦点项
- `Escape` 关闭并将焦点返回触发器
- `Tab` 离开控件时自动收起下拉框

焦点现在会在 trigger、search input 与 option 之间进行明确管理，避免失焦与不可预期的跳转。

### 5. 补齐可访问名称与基础 ARIA 语义

当前控件已经补齐：

- `aria-haspopup="listbox"`
- `aria-expanded`
- `aria-controls`
- `role="listbox"`
- `role="option"`
- `aria-selected`
- trigger、listbox、search input 的明确可访问名称

### 6. 补成更完整的 ARIA 分组版本

本轮进一步把原来的“视觉标题”升级成了真正的语义分组。

也就是说，`Official Topics` 与 `Your Saved Topics` 不再只是用户看得到的标题文字，而是以 `role="group"` + `aria-labelledby` 的方式暴露给辅助技术。这样一来，屏幕阅读器可以理解这些选项分别属于哪个分组，而不是把它们当成一串没有上下文的平铺内容。

### 7. 修复重新打开时的选中态同步问题

现在，在用户选中某个题目后重新打开下拉框时，选项列表会重新渲染，确保：

- 视觉上的 `.selected` 状态正确
- 语义上的 `aria-selected="true"` 正确

这样视觉状态与辅助技术读到的状态就能保持一致。

## 什么是 ARIA 分组

ARIA 分组的意义在于：把“视觉上已经存在的分组信息”同步提供给辅助技术。

在本界面中，`Official Topics` 与 `Your Saved Topics` 本来就代表两类来源不同的话题。如果没有 ARIA 分组，屏幕阅读器用户听到的通常只是一串选项文本；加入 ARIA 分组后，辅助技术可以理解这些选项属于两个不同的有名称的分组。

这意味着用户不仅能知道“有哪些选项”，还能知道“这些选项来自哪个类别”。

## 结论

Free Practice 输入区在这一轮优化后，已经在以下四个方面获得了明显改善：

1. 更容易理解
2. 更容易浏览和搜索
3. 更容易阅读和区分状态
4. 具备可接受的键盘与辅助技术访问基础

按当前自定义下拉框设计，这一版实现已经可以作为可交付版本使用。

## 后续可选优化

如果继续迭代，一个自然的下一步是进一步补充更细粒度的辅助技术提示，例如：

- 分组内选项数量提示
- 搜索结果数量反馈
- 更细的 active descendant 模式增强

这些都属于增量优化，不影响当前版本的可用性与完整性。
