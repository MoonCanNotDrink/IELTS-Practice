## 1. 概要

### 项目定位

IELTS Speaking AI 是一个面向 IELTS 口语（及写作）训练的全栈练习平台。覆盖完整模考流程（Part 1 → Part 2 → Part 3）、自由练习、动态追问、语音播报（TTS）、转写（ASR）、多维评分反馈和历史复盘。

### 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | 原生 HTML / JavaScript / CSS（无框架） |
| 后端 | FastAPI / Python（异步） |
| 数据库 | SQLite（本地）/ PostgreSQL（Cloud Run） |
| 语音转写 | Azure Speech SDK + faster-whisper |
| 音频分析 | librosa |
| 大模型 | Gemini（评分 + 追问生成） |
| 认证 | JWT + bcrypt |
| 部署 | Docker / Cloud Run |

### 总体评级

| 维度 | 评级 | 说明 |
|------|------|------|
| 功能完整度 | ⭐⭐⭐⭐ | 核心流程完整，写作功能已上线 |
| UI/UX | ⭐⭐⭐⭐ | 设计系统成熟，主题系统完善 |
| 代码质量 | ⭐⭐⭐ | 功能正确但存在大量重复代码 |
| 文档 | ⭐⭐⭐ | 双语文档已有体系，但有孤立文件和不一致 |
| 测试 | ⭐⭐ | E2E 测试覆盖核心流程，但无 CI/CD 自动运行 |
| DevOps | ⭐⭐ | Docker 部署可用，缺 CI/CD pipeline |
| 安全 | ⭐⭐ | 认证框架存在，但配置不够安全 |

## 2. 功能设计与实现

### 2.1 已实现功能清单

#### Speaking 模块
- ✅ 完整模考流程：Part 1（3 题）→ Part 2（准备 + 作答）→ Part 3（3 题）
- ✅ 自由练习模式：题库选题、自定义题目、自定义作答时长
- ✅ 动态追问：Gemini 生成 follow-up questions（Part 1 & Part 3）
- ✅ 语音考官：Azure TTS 播报题目
- ✅ 录音与转写：MediaRecorder + faster-whisper / Azure Speech
- ✅ 多维评分：流利度、词汇、语法、发音、连贯性
- ✅ 发音评估：Azure Pronunciation Assessment
- ✅ 收藏题目：保存/取消保存自由练习题目
- ✅ 全面评分模式：带详细反馈的 full-exam scoring

#### Writing 模块
- ✅ 写作练习：Task 1 / Task 2 prompt 生成
- ✅ 写作评分：Gemini 评分 + 详细反馈
- ✅ 写作历史：尝试记录与查看

#### 通用功能
- ✅ 用户认证：注册 / 登录（JWT + 邀请码）
- ✅ 历史记录：统一仪表盘（口语 + 写作）
- ✅ 趋势图表：得分趋势可视化
- ✅ 深色 / 浅色 / 系统主题
- ✅ 多用户数据隔离

### 2.2 功能设计优点

1. **流程设计贴近真实考试**：Part 1 → Part 2 → Part 3 的状态机设计（`home → topic → part1 → part2prep → part2speak → part3 → scoring`）准确反映 IELTS 考试结构。

2. **自由练习灵活度高**：支持从题库选题、自定义题目、自定义作答时长，满足不同水平用户需求。

3. **动态追问设计合理**：使用 Gemini 生成 follow-up questions，模拟真实考官追问行为，增强练习价值。

4. **评分维度全面**：涵盖流利度、词汇、语法、发音、连贯性等 IELTS 评分标准。

### 4.3 数据模型

**ORM 模型（`models.py`，164 行）**

模型设计合理，覆盖了所有业务需求：
- `User`：用户基本信息 + 密码哈希
- `Topic`：题目（含 Part 1/2/3 配置）
- `PracticeSession`：练习会话（含完整状态跟踪）
- `Recording`：录音记录（含转写文本、评分）
- `SavedTopic`：收藏题目
- `WritingPrompt` / `WritingAttempt`：写作题目与尝试

不足：
- 缺少数据库迁移工具（Alembic）— 依赖 `create_all()` 自动建表
- 无索引优化声明（大量查询按 `user_id` 过滤，但未显式建索引）

## 8. 安全

### 8.1 问题

| # | 问题 | 严重度 |
|---|------|--------|
| SEC-1 | **CORS 设置为 `allow_origins=["*"]`** — 允许任意域名跨域请求 | 高 |
| SEC-2 | **默认 JWT 密钥硬编码**（`config.py` 中 `JWT_SECRET` 有默认值） | 高 |
| SEC-3 | **默认邀请码硬编码**（`config.py` 中 `INVITE_CODE` 有默认值） | 中 |
| SEC-4 | `dev.py` 的 reset endpoints 在生产环境可访问 | 高 |
| SEC-5 | 无 rate limiting（登录、注册、API 均无频率限制） | 中 |
| SEC-6 | JWT token 无刷新机制（仅有过期时间） | 低 |

### 8.2 安全优点

- ✅ 密码使用 bcrypt 哈希存储
- ✅ JWT 鉴权中间件保护 API 路由
- ✅ 用户数据隔离（按 `user_id` 过滤查询）
- ✅ 注册需要邀请码（虽然默认值硬编码）
