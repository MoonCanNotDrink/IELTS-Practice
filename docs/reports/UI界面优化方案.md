# IELTS Speaking AI 前端界面与布局优化方案

## 1. 现状分析

通过对项目现有 `frontend/index.html`、`frontend/history.html` 和 `frontend/index.css` 的分析，我们发现原有的 UI 设计呈现出以下特点与局限性：

### 1.1 优点
- **视觉基调明确**：采用了现代感较强的“深色毛玻璃 (Glassmorphism)”风格，提供了多套主题（深色/浅色/跟随系统）。
- **组件化程度较高**：通过 CSS 变量系统定义了核心色彩、字体、阴影、圆角等设计 Token。
- **信息层次清晰**：主要功能以卡片（Card）形式划分，功能区域分明。

### 1.2 存在的问题与优化空间
1. **布局受限**：主容器 `.app-container` 的 `max-width` 设定为 `720px`，这在移动端表现良好，但在大屏幕桌面上显得过于狭窄，利用率不高。
2. **按钮组展示臃肿**：首页模式选择区域的按钮组采用了 `flex-direction: column` 加上 100% 宽度，这使得桌面端的按钮呈现巨大块状，视觉上显得比较笨重。
3. **视觉细节不够精致**：卡片的毛玻璃效果虽然有，但边框及高光（Highlight）的处理不够立体，缺乏物理厚度感。
4. **色彩与动画**：颜色对比度可适当微调，按钮悬浮效果可以通过改变亮度、增加内部发光来提升质感。

---

## 2. 优化方案设计

为了提升用户体验（UX）和视觉美感（UI），我们计划实施以下优化策略：

### 2.1 布局扩展 (Layout Expansion)
- 将主容器的最大宽度（`max-width`）从 `720px` 扩展至 `800px`，同时微调左右的 `padding`，让整个界面在桌面端呼吸感更强，且不影响移动端适配。

### 2.2 重构按钮组交互 (Button Grid Refactoring)
- 移除原本强制堆叠且全宽的 `.btn-group`。
- 引入响应式网格布局 `.btn-grid`。利用 `grid-template-columns: repeat(auto-fit, minmax(240px, 1fr))`，在宽屏下呈现为并排排列（两列或三列），在窄屏下自动回落为单列堆叠，极大提升桌面端的美观度。
- 为不同模式（如“Speaking”与“Writing”）增加精致的色彩标识线（Color Badge），引导视觉焦点。

### 2.3 升级卡片与毛玻璃质感 (Enhanced Glassmorphism)
- **卡片深度**：为 `.card` 增加内部顶部高光阴影 `inset 0 1px 1px rgba(255, 255, 255, 0.08)`，模拟光线折射，增加物理厚度。
- **模糊度与发光**：将背景模糊度提升至 `blur(24px)`，在悬浮时增加阴影扩散范围，使交互更具动态生命力。
- **色彩调整**：背景色微调至深邃的 `#090c15`，以更好地突显毛玻璃模块。

### 2.4 提升字体版式 (Typography Enhancements)
- 加大主标题（`h1`）的字号至 `2.25rem`，并进一步优化标题的渐变渲染与居中对齐。
- 微调各类小文本（`.subtitle`，类别标题）的字母间距（`letter-spacing`），使其更具现代排版感。

### 2.5 数据展示优化 (History & Score Grid)
- 历史记录页面的分数面板增强悬浮交互，整体分数（`.overall`）添加轻微背景渐变及高光框，使其成为视觉焦点。
- 重构打分记录和成绩单的背景色透明度，降低视觉刺眼度。

---

## 3. 实施路径
1. **更新全局 CSS 变量**：调整颜色、阴影半径等基础 Token。
2. **修改组件样式**：更新 `.card`, `.btn`, `.app-container`, `.score-item` 等类。
3. **HTML 结构微调**：将首页 `index.html` 中的按钮列表重构为 `<div class="btn-grid">`，并加入修饰性的区块标题。
4. **适配测试**：在各类视口尺寸（Desktop/Tablet/Mobile）下确认响应式表现。
